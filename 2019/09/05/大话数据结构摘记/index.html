基本概念与术语

  • 数据

    描述客观事物的符号, 计算机中可以操控, 可以被识别的对象, 也是所有能输入给计算机处理的符号集合.

  • 数据元素

    组成数据的, 有一定意义的基本单位, 在计算机中通常作为整体处理.

    (如在人类中数据元素就是人, 畜类中就是牛, 马等)

  • 数据项

    一个数据元素由若干个数据项组成, 它是数据不可分割的最小单位

  • 数据对象

    是性质相同的数据元素的集合, 数据的子集. 性质相同的定义为, 具有相同数量和类型的数据项的数据元素.

    (实际应用中处理好的数据元素通常具有相同性质, 为了不产生混淆, 我们都将数据对象简称为数据)

  • 数据结构

    一般的, 不同数据元素之间不是独立的, 而是存在特定的关系, 我们称这种关系为结构.

    而数据结构, 是相互之间存在一种或多种特定关系的数据元素的集合.

逻辑结构与物理结构

逻辑结构

逻辑结构是数据对象中数据元素之间的互相关系. 基本分为四种:

  1. 集合结构 (例: Set )

    集合结构中的数据元素除了同属一个集合外, 他们没有任何其他关系, 类似于数学中的集合.

  2. 线性结构 (例: LinkedList )

    线性结构中, 数据元素之间是一对一的.

  3. 树形结构 (例: TreeMap )

    存在一对多的层次关系

  4. 图形结构

    类似于图论中的图, 地图上地点之间的关系. 它可以是有向的也可以是无向的.

物理结构

物理结构是数据的逻辑结构在计算机中的储存形式:

顺序储存结构

把数据元素存放在地址连续的储存单元里, 其数据间的逻辑关系和物理关系是一致的,

链式储存结构

把数据元素存放在任意的储存单元里, 它们可以是连续的, 也可以是不连续的

抽象数据类型 (ADT)

抽象:

抽取出事物具有的普遍性的本质.

数据类型:

一组性质相同的值的集合及定义在此集合上的一些操作的名称.
在大多数编程语言中, 数据类型都可以分为两种:

  1. 原子类型: 是不可以再分解的基本类型, 包括整型, 实型, 字符型等.
  2. 结构类型: 由若干个类型组合而成, 可以再分解, 如若干个整形数据组成的数组.

抽象数据类型:

ADT (Abstract Date Type) 是指一个数学模型, 与定义在该模型上的一组操作.
它的建立体现了程序设计中, 问题分解, 抽象和信息隐藏的特性.
