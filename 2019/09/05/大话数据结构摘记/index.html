基本概念与术语

  • 数据

    描述客观事物的符号, 计算机中可以操控, 可以被识别的对象, 也是所有能输入给计算机处理的符号集合.

  • 数据元素

    组成数据的, 有一定意义的基本单位, 在计算机中通常作为整体处理.

    (如在人类中数据元素就是人, 畜类中就是牛, 马等)

  • 数据项

    一个数据元素由若干个数据项组成, 它是数据不可分割的最小单位

  • 数据对象

    是性质相同的数据元素的集合, 数据的子集. 性质相同的定义为, 具有相同数量和类型的数据项的数据元素.

    (实际应用中处理好的数据元素通常具有相同性质, 为了不产生混淆, 我们都将数据对象简称为数据)

  • 数据结构

    一般的, 不同数据元素之间不是独立的, 而是存在特定的关系, 我们称这种关系为结构.

    而数据结构, 是相互之间存在一种或多种特定关系的数据元素的集合.

逻辑结构与物理结构

逻辑结构

逻辑结构是数据对象中数据元素之间的互相关系. 基本分为四种:

  1. 集合结构 (例: Set )

    集合结构中的数据元素除了同属一个集合外, 他们没有任何其他关系, 类似于数学中的集合.

  2. 线性结构 (例: LinkedList )

    线性结构中, 数据元素之间是一对一的.

  3. 树形结构 (例: TreeMap )

    存在一对多的层次关系

  4. 图形结构

    类似于图论中的图, 地图上地点之间的关系. 它可以是有向的也可以是无向的.

物理结构

物理结构是数据的逻辑结构在计算机中的储存形式:

顺序储存结构

把数据元素存放在地址连续的储存单元里, 其数据间的逻辑关系和物理关系是一致的,

链式储存结构

把数据元素存放在任意的储存单元里, 它们可以是连续的, 也可以是不连续的

抽象数据类型 (ADT)

抽象:

抽取出事物具有的普遍性的本质.

数据类型:

一组性质相同的值的集合及定义在此集合上的一些操作的名称.
在大多数编程语言中, 数据类型都可以分为两种:

  1. 原子类型: 是不可以再分解的基本类型, 包括整型, 实型, 字符型等.
  2. 结构类型: 由若干个类型组合而成, 可以再分解, 如若干个整形数据组成的数组.

抽象数据类型:

ADT (Abstract Date Type) 是指一个数学模型, 与定义在该模型上的一组操作.
它的建立体现了程序设计中, 问题分解, 抽象和信息隐藏的特性.

树

Tree 是有 n 个结点的有限集, n=0 时未空树, 任意一棵非空树中, 只有一个根结点,

结点相关术语

  • 度 : 结点拥有的子树数目被称为结点的度 (Degree)
  • 结节点 : 度为 0 的结点, 被称为Leaf, 或者终端结点
  • 分支结点 : 度不为 0 的结点
  • 树的度 : 度不为 0 的结点内, 各结点的度的最大值

结点间关系

  • Parent: 父结点
  • Sibling: 兄弟结点
  • Child: 子结点

其他概念

  • Level: 层级从根开始定义, 根结点也就是第一层

顺序表下的实现方法

父结点表示法

每个结点中, 加入指向其他结点的下标

下标 data parent rightsib
0 A -1 -1
1 B 0 2
2 C 0 -1

其中rightsib是指向右兄弟的下标, 用于快速同级遍历, 这个可以选择实现

子结点表示法

每个结点中除了数据部分, 还保存着一个链表的头指针, 这个链表中保留着这个结点的, 所有子节点的下标.

子节点表示法的结点中, 除了数据部分, 还可以加入一个parent部分, 来保存这个结点的父节点引用, 以便回溯.

孩子兄弟表示法

对于树, 有这样一个事实: “任意一棵树中, 结点的第一个孩子如果存在, 那么就是唯一的, 右兄弟如果存在, 那么就是唯一的.” 因此, 每个结点设置两个指针, 分别指向结点的第一个孩子, 和该结点的右节点即可.

data firstchild rightsib

如果有必要的话, 我们还可以增加一个 parent 域, 用于快速回溯.

二叉树

递归定义

Binary Tree 是由n个结点的有限集合, 该集合或者为空集, 或者一个根结点和两棵互不相交的, 分别称为根结点的左子树和右子树的二叉树组成的

二叉树特性

  • 每个结点的度最多为2
  • 左右子树是有顺序, 不可交换
  • 即使只有一颗子树, 也要区分它是左子树还是右子树

特殊的二叉树

  • 斜树

    只有左子树 或者 只有右子树, 这是退化成链表的一种树结构

  • 满二叉树

    所有分支结点都存在左子树和右子树, 并且所有的叶子都在同一层上,

  • 完全二叉树

    二叉树未满, 但是二叉树上结点 i 的位置, 与满二叉树上对应的 i 位置完全相同

    可以认为, 完全二叉树是满二叉树的超集

二叉树性质

  1. 二叉树的第 i 层至多有 $2^{i-1}$ 个结点

  2. 深度为k的二叉树最多有 $2^k-1$ 个结点,

    由等比数列求和公式 $S_{n}=a_{1}*{\frac{1-q^n}{1-q}}$ 得到

  3. 对于任意一棵二叉树, 若叶子结点 (终端结点) 的数目为 $n_0$, 度为 2 的结点数位 $n_2$, 那么
    $$
    n_0=n_2+1
    $$

  4. n 个结点的完全二叉树的深度为
    $$
    \lfloor log_2n \rfloor +1
    $$
    这里$\lfloor x\rfloor$表示不大于x的最大整数. 这个性质可有性质 2 推导而来

  5. 对于一棵有n个结点的完全二叉树, 结点按层编号, 每层从左到右, 对任意结点 i 有

    • 如果 $i=1$ , 则结点 $i$ 是二叉树的根, 无父节点

    • 如果 $i>1$ , 其父节点下标 = $\lfloor i/2 \rfloor $

    • 如果 $2i>n$ , 结点 $i$ 无左子结点,

      否则左子结点是结点 $2i$

    • 如果 $2i+1>n$ , 结点 $i$ 无右子结点,

      否则其右子结点 $2i+1$

二叉树储存结构

顺序储存结构

如果一棵二叉树是完全二叉树, 那么这颗二叉树可以用无空隙的顺序储存结构表示

完全二叉树的这种能被顺序存储的特性使得我们的堆排序可以被应用.

对于一般的二叉树, 我们将空的结点位置跳过即可, 但如果树中存在多个斜树时, 顺序储存将会浪费大量的存储空间.

链式存储结构

由于顺序存储结构只适用于完全二叉树, 我们一般使用链式结构实现二叉树存储

lchild data rchild
/ / /

和树一样, 如果有需要, 我们可以为结点加上 parent 域, 形成三叉链表, 来方便回溯

二叉树的遍历

  • 前序遍历, 深搜, 每个结点先左后右

  • 中序遍历, 深搜

    遇到任意一个父节点时, 逻辑均为, 左子节点 -> 父节点 -> 右子结点

    从树的最左侧叶子结点开始遍历, 代码中的体现是, 将左结点的递归函数挪到do_something()之前, 这也体现了中遍历的中的意思

  • 后序遍历, 深搜

    遇到任意一个父节点时, 逻辑均为, 左子结点 -> 右子结点 -> 父节点

    与名字的意思一样, 后序遍历就是将do_something()挪到两个结点递归函数之后

  • 层序遍历, 广搜, 逐层从左到右

通过遍历序列推导遍历结果

若已知 前序遍历序列和中序遍历序列 或者 后序遍历序列和中序遍历序列, 则可唯一确定一棵二叉树. ( 已知前序和后续无法得到二叉树 ).

遍历序列的解析中, 通过 前序和后序 遍历, 我们可以确定根结点位置, 再将根结点位置对照中序遍历序列, 可以得到子树与结点的结构关系.

建立二叉树

如果需要建立上述结构的二叉树, 我们输入中也要指明空指针, AB#D##C##

线索二叉树

普通的链式存储二叉树, 每个叶节点上都存在两个空指针, 在斜子树的结点上也存在一个或多个空指针, 我们可以利用这些空位来构建线索二叉树, 实现二叉树结点的前驱和后继关系.

将二叉树以某种次序遍历使其变为线索二叉树的过程, 称为线索化
一般, 我们使用中序遍历来线索化二叉树.

为了线索化, 我们需要对叶子和斜树结点的lchild和rchild与普通结点的进行区分, 我们需要为结点添加两个Boolean标识位(IsThread / IsTag), 用于标识这个结点是不是用于指示前驱和后继的结点.

中序遍历线索化实现

将中序遍历中的do_something()改为线索化代码即可, 使用中序遍历搭建链表可以使链表和二叉树的关系更直观.

线索二叉树中的头节点

注意到, 如果按照上一个 code snip 中那样做, 我们又会得到两个空指针, 分别位于序列化之后的第一个结点和最后一节结点, 我们可以创建一个头节点, 将这两个指针指向我们的头节点.

同时头节点应该含有

  1. 指向根结点的引用.
  2. 指向线索化二叉树的第一个结点引用.
  3. 指向线索化二叉树的最后一个结点的引用.
