OpenGL -> Open Graphics Library

状态机

OpenGL采用状态机模式, 一系列变量用来描述 OpenGL 此刻如何运行, 其状态被称为 OpenGL Context, .例如通过改变上下文变量, 我们可以改变OpenGL状态, 从而告诉OpenGL如何绘制图像. 这种特性使得OpenGL中有两大函数, 状态设置函数(State-changing Function), 以及状态使用函数(State-using Function).

对象

OpenGL的内核是一个C库, 所以为了便于开发, OpenGL也引入了一些抽象层, 对象(Object)就是其中一个

GLFW, OpenGL库, GLAD

  • GLFW

    一个轻量级的, 开源跨平台, 支持 OpenGL 和 OpenGL GS 的 Library, 用来为OpenGL提供窗口管理, 读取输入, 处理事件的功能

  • OpenGL库

    windows下 opengl32.lib 包含在 Microsoft SDK 中, 在VS安装时就被默认安装

  • GLAD

    由于OpenGL驱动版本众多, 如果不使用GLAD, 我们需要自己在运行时查询函数

    看上去就像这样

    通过使用GLAD提供的服务, 我们只需要定义OpenGL的版本和对应的语言, 它就会帮我们加载所有对应的 OpenGL 函数

    实例化GLFW窗口

  • glfwInit()函数用于初始化 GLFW
  • glfwWindowHint()用于配置 GLFW的内容, 这里设置了版本, 并且设置OpenGL为核心模式

创建渲染窗口的流程

  1. 使用glfwCreateWindow()创建窗口对象window
  2. 设置window为主线程的上下文对象glfwMakeContextCurrent()
  3. 通过 GLFW 获取本机opengl函数指针的地址, 初始化 GLAD
  4. 通过 GLAD 设置渲染窗口的尺寸大小glViewport(), 并通过 GLFW 注册窗口大小变化时的回调函数
  5. 设置一个控制渲染刷新的循环体, 如果窗口不必关闭, 不断执行相关渲染指令

三角形, 顶点, 渲染管线

  • 顶点数组对象: Vertex Aarray Object, VAO
  • 顶点缓冲对象: Vertex Buffer Object, VBO
  • 索引缓冲对象: Element Buffer Object, EBO 或 Index 开头的 IBO

OpenGL着色语言: OpenGL Shading Language, GLSL

图形渲染管线接受一组3D坐标, 转化为屏幕上的2D有色像素输出, 蓝色的阶段我们可以注入自定义的着色器

简易渲染流程

  1. 以数组的形式传入三个顶点数据, 它们可以简单地只包括3D位置和颜色值, 但也可以包含任何我们想用的数据

    顶点: Vertex, 顶点属性: Vertex Attribute

  2. 顶点着色器: 见顶点输入部分

  3. 图元装配: 这里我们需要为OpenGL指定我们要装配的图元类型, 分别是GL_POINTS, GL_TRIANGLES, GL_LINE_STRIP, 分别装配为点, 三角形, 线

  4. 几何着色器: 将传递进来的一系列顶点的集合作为输入, 产生新顶点构造出新的图元

  5. 光栅化阶段: 将图元映射为最终屏幕上相应的像素, 生成片段, 并执行裁切去掉超出视图的部分

    OpenGL中的片段是渲染一个像素所需的所有数据

  6. 片段着色器: 计算一个像素的最终颜色, 它也包含着3D场景中的数据 如光照, 阴影, 光的颜色等, 这是所有OpenGL高级效果产生的地方.

  7. Alpha测试和混合: 检测对应的深度值判断是否丢弃, 同时检测 Alpha 值用于对颜色进行混合

顶点输入

OpenGL仅当3D坐标在标准化设备坐标内时才会去处理它, 所以我们输入的有效顶点数据应该处于 [-1, 1] 之间, 之后它们会根据glViewport()函数提供的数据变化为屏幕空间坐标.

定义了顶点数据后, 将它发送给图形渲染管线的第一个阶段: 顶点着色器, 它于GPU上创建内存用于储存顶点数据, 并配置OpenGL如何解释这些内存, 指定如何发送给显卡. 接着它会处理我们在内存中指定数量的顶点.

这里用于储存顶点的内存被顶点缓冲对象VBO管理着, 它会在GPU内存中储存大量顶点, 下面是使用VBO的步骤

  1. 使用glGenBuffers()和一个缓冲ID生成一个 VBO 对象

  2. 使用glBindBuffer()定义 VBO 对象的缓冲类型, 这里GL_ARRAY_BUFFER是指顶点缓冲

  3. 使用glBufferData()将顶点数据转移到缓冲的内存中

    第四个参数有三个等级形式:

    1. GL_STATIC_DRAW: 数据不会或几乎不变
    2. GL_DYNAMIC_DRAW: 数据会被改变很多
    3. GL_STREAM_DRAW: 数据每次绘制都会改变

    给定的等级越高, 显卡就倾向于将其放在能高速写入的内存部分

    创建顶点着色器

    现代OpenGL至少需要设置一个顶点和片段着色器, 需要使用GLSL (OpenGL Shading Language) 编写顶点着色器
