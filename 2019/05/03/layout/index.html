Android UI 的声明与管理概述

Android 的 UI 控件由两种方法声明和设置

  • 于 XML 中声明 UI 元素, 利用 Android 提供的对应于View类的简明XML词汇, 构建默认界面
  • 运行时实例化布局元素, 在编程中创建 View 对象和 ViewGroup 对象, 并操作其属性

Android 的这种设计使得应用的外观更好地于控制应用行为的代码隔离, 调整外观时, 无需修改源代码, 并且可以根据不同的屏幕建立不同的 XML 布局

ID

如果对一个 View 对象在 XML 文档中添加 id 标签, 它将会被作为整型引用, 被解析到 R.java 文件中, 即在结构树中对 View 对象进行唯一标识.

在 XML 文档中, 它总是这样的

字符串开头处的 @ 用来指示 XML 解析程序应解析并展开 ID 字符串之后的部分,
加号 ( + ) 标识这是一个新的资源名称

XML属性 说明
android:drawable 放一个drawable资源
android:state_pressed 按下状态，如一个按钮触摸或者点击。
android:state_focused 取得焦点状态，比如用户选择了一个文本框。
android:state_hovered 光标悬停状态，通常与focused state相同，它是4.0的新特性
android:state_selected 选中状态
android:state_enabled 能够接受触摸或者点击事件
android:state_checked 被checked了，如：一个RadioButton可以被check了。
