定义自定义特性

良好的自定义View应该允许在 XML 文档里对其特性进行修改,
为了实现这个功能, 应该实现以下四点

  • 在资源文件中添加<declare-styeable>标签, 并为其添加内容
  • 为所需要的特性设定特殊的值
  • 在代码运行时取回所需的值
  • 将取回的值应用到代码中

一般会将<declare-styeable>标签添加在 res/values/attrs.xml 文件中. 这里提供谷歌官方 attrs.xml 示例

自定义的特性和官方控件与布局的特性属于不同的命名空间

官方特性存在于 http://schemas.android.com/apk/res/android 命名空间
自定义特性存在于 http://schemas.android.com/apk/res/[your package name] 命名空间

提供代码调整属性的方法

使用特性已经为我们提供了一种强大的控制我们自定义View的行为和外观的方法, 但是他们只能在视图创建时被读取一次, 为了实现动态的参数调整, 我们需要为我们的属性添加getter于setter方法, 用于在代码中主动地动态调整他们的状态. 提供谷歌官方示例

每当调整了view的外观之后都该依次调用这两个方法

  • requestLayout()

重写 onDraw(Canvas canvas)

自定义view的 UI 绘制步骤被限制在onDraw方法中, 它传入一个Canvas对象, Canvas对象中提供了很多绘制基本图形的方法,

Android 图形框架将绘制分为两个类

  • Canvas 定义要在屏幕上绘制什么图形
  • Paint 定义怎么绘制图形, 如定义颜色, 风格, 字体, 填充等等
