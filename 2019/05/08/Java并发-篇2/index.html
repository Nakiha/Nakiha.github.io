定义进程与线程

Process 进程

进程是计算机程序关于某数据集合上的一次运行活动, 是系统进行资源分配和调度的基本单位, 是操作系统结构的基础.

在早期面向进程设计的计算机结构中, 进程是程序的基本执行实体; 在当代的面向线程设计进程是线程的容器, 程序是指令数据及组织形式的描述, 进程是程序的实体.

在 Windows 中任务管理器看到的一个个实体就是一个个活的进程

Thread 线程

线程是轻量级的进程, 是程序执行的最小单位.

使用多线程而不是多进程进行并发程序的设计, 主要是因为线程切换和调度的成本远远小于进程.

一个进程中会包含多个线程, 通过多个线程的协作来完成整个 “任务”.

线程的生命周期

线程的所有状态都在Thread中的State中枚举定义

这六个状态也就组成了线程的生命周期

线程的基本操作

新建线程

  • start() 会开启一个新线程
  • run() 在当前线程中执行对应线程的代码

给线程添加 “任务”

重载 run()

这里使用了匿名内部类, 继承了Thread, 通过将run重载把任务填进去

但 Java 是单继承的, 这意味着继承是一种很宝贵的资源, 可使用接口来达到同样的效果

使用 Runnable 接口

在构造Thread类时可以将传递一个Runnable接口来定义任务, 从而避免继承的浪费.

停止线程

曾经用于强制停止一个线程的stop()已被废弃, 它会在调用时直接终止线程, 立即释放这个线程所持有的锁, 这经常会导致一致性错误. 所以, 应该由我们自己来决定如何何时停止线程

例如, 可以通过设置与检测标志符来停止一个线程

插入话题 volatile 关键字

将一个变量声明为volatile后, 编译器会注意到这个变量是共享的, 因此不会将这个变量上的操作与其他内存操作一起重排序. volatile变量不会被缓存在寄存器或者对其他处理器不可见的地方

而对非volatile变量进行读写的时候，每个线程先从内存拷贝变量到CPU缓存中。如果计算机有多个CPU，每个线程可能在不同的CPU上被处理，这意味着每个线程可以拷贝到不同的 CPU 缓存中

访问volatile变量不需要加锁, , 它是一种比synchronized更轻量级的同步机制
