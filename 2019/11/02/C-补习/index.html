标志 功能
boolalpha 可以使用单词”true”和”false”进行输入/输出的布尔值.
oct 用八进制格式显示数值.
dec 用十进制格式显示数值.
hex 用十六进制格式显示数值.
left 输出调整为左对齐.

internal 将填充字符回到符号和数值之间.
uppercase 以大写的形式显示科学记数法中的”e”和十六进制格式的”x”.

  • C++ 销毁对象

    new 的对象必须手动销毁, 使用delete销毁基本类型和基本类型的数组, 使用delete[]销毁new出来的指针数组

    如果是局部变量, 过了作用域之后, 它们会自动析构

  • 函数中 const 的位置

    函数前面为const意味着返回值是只能被读取的, 需要用const变量去承接

    函数后面的const意味着这个函数不可以修改除了mutable之外的类成员

  • C++ 11/14 constexpr

    使用constexpr修饰的函数, 如果向它传递的是常量值或者其他constexpr的函数, 那么编译期就会得到这个结果, 运行时直接会使用结果而不是函数.

  • 初始化列表

  • C++虚函数

    一个类如果有虚函数, 它的虚函数会储存在这个类的最前方, 直接影响sizeof返回的结果, 如果使用override覆写虚函数, 那么这个函数必须重载同名函数, 否则过不了编译, 增强了健壮性

  • C++对齐

    以最长的变量为对齐长度, 空类的大小为1字节, 这是为了储存地址

  • new 和非 new 创建对象

    new的对象创建在 heap 上, 需要手动 delete 或者 free
    非 new 的对象创建在 stack 上, 快, 但是资源有限, 如果创建过多会导致栈溢出

  • NULL 和 nullptr

    C++ 11 引入的 nullptr 用于区分 指针和整数 的空状态, nullptr是一个C++关键字

  • decltype 类型推断

    使用auto时, 必须有一个已经赋值完成的该类型实例, decltype可以只推断类型, 不使用其值

  • 工厂模式下的析构函数

    由于工厂模式下一般是用父类指针去引用子类对象, 由于工厂模式的特点, 其子类对象只能动态创建, 储存在堆中, 我们需要给父类的析构函数定义为virtual这时, 使用delete时, 我们才能正确地调用到子类的析构函数来正确析构

  • 析构函数异常

    析构函数不应该抛出异常, 如果一个析构函数需要调用一个会抛出异常的函数, 那么应该在析构函数内捕获这个异常, 同时提供一个不会捕获这个异常的函数成员, 提供给用户手动调用到这个函数, 给予用户手动处理异常的机会

  • 纯虚函数与抽象类

    C++ 中带有virtual关键字, 并且赋为 0 的函数成员为纯虚函数, 带有纯虚函数的类被称为抽象类, 只能作为派生类为子类服务, 如果子类没有实现所有的纯虚函数, 那么它也会变成抽象类.

  • 纯虚函数与构造和析构函数

    C++ 在构造一个子类时需要先构造其父类, 如果父类的构造函数中调用了一个虚函数, 由于子类尚未构建, 无法获得子类实现的这个虚函数的定义, 这会导致程序终止. 如果确实需要在构造时调用一个函数, 并且在不同子类时行为不同, 需要使用 一个带参数成员函数, 并由父类的构造参数来传参, 来完成这个过程

  • 自定义操作符时, 使返回的值符合逻辑

  • 自动指针 auto_ptr<> 以对象管理资源

    工厂模式下, 期待用户手动调用 delete 是错误的, 我们可以利用 C++ 标准库下的自动指针模板类, 来管理这个对象, 自动指针类出作用域时, 自动调用它管理的类的析构函数

流状态 iostate

iostate 是一种 bitmask 类型, 它由三个 ios_base 元素组成, eofbit, badbit, failbit. 分别标识, 达到文件尾, 流是否被破坏, 输入操作未能获得预期字符, 或者输出操作没有写入预期字符

  • good()

    如果流的所有位都被清除, 返回true

  • clear()

    默认清除所有位, 可以指定哪些位不被清除
    如果只是为了跳过错误的字符, 还需要手动使用cin.get()等函数来清除输入队列

  • setstate()

    设置指定位

指定流处于某个状态时为异常

为流调用 exceptions() 函数, 传入对应的流状态( 如: ios_base::failbit ), 当流处在这个状态时, 将抛出一个ios_base::failure异常

open(,)的第二个参数

ofstream和ifstream都为open()的实现提供了默认参数, 分别是ios_base::in和ios_base::out | ios_base::trunc`,

常量 含义
ios_base::in 打开文件, 读取
ios_base::out 打开文件, 写入
ios_base::ate 打开文件, 移到文件尾
ios_base::app 追加到文件尾
ios_base::trunc 如果文件存在, 截短文件
ios_base::binary 二进制文件

注意: 由于二进制文件不存在换行之类的字符, 必须使用 ios_base::binary 来open二进制文件, 对于二进制文件, 使用指定数据大小的 write 和 read 来输入输出,
