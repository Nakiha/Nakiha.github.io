C#特性学习

Posted on 2019-08-30 | Edited on 2019-09-08

本篇文章内容参考自 “Illustrated C# 2012”
二级标题格式: <内容> <页码>

引用参数 63

C#允许像C++一样传递值的引用, 但默认情况下与Java一样是值传递

如果要传递引用参数, 应使用ref关键字

输出参数 68

为参数添加out关键字, 意味着这个参数是用来被赋值的, 而不是读取.

参数数组 70

允许零个或多个实参相同类型形参, 对应到一个被params关键字修饰的同类型数组

命名参数 75

在追求强自描述代码时, 可以使用形参的名字来描述实参

类成员 - 属性 91

属性不为数据储存分配内存, 它像是一个函数成员, 通过其set和get访问器来更改类中的其他字段. 通过设置访问器的可见程度与选择性实现set和get可以更简单明确地完成代码的封装

  • set访问器

    拥有一个隐式的值参, 名称为value用来代表传入的值

  • get访问器

    必须return一个符合类型的值

使用属性

get和set访问器不能直接被调用, 使用属性就和使用普通字段一样

字段修饰符 - readonly

readonly与const作用相似, 但const必须在字段声明时初始化, 即编译时决定, readonly只需在任意构造函数中进行设置

索引器

索引器可以将相同类型的字段进行索引, 这将有助于部分代码的编写, 它有是一组get和set访问器,

索引器常常和swich语句结合使用, 下面给出一个set的例子

分部方法

分布类允许使用partial关键字分离出类中的可覆写部分和不可覆写部分

派生类屏蔽基类成员

new关键字的使用可以屏蔽基类的同名成员, 如果要访问基类方法, 所有派生类中都有base隐式地指向其基类.

派生类覆写基类成员

如果要得到 Java 中的重载逻辑, 就需要使用virtual和override关键字, 基类方法用virtual标注为虚方法.

构造函数初始化语句

C#允许构造函数指定一个构造函数初始化语句, 可以理解成Java中通过语法糖分离了在块中的super.constructr();.

自定义类型转换

implicit operator和explicit operator分别进行隐性类型转换与显式类型转换

运算符重载

单个operator关键字用于运算符重载

Lambda表达式

C#的Lambda表达式用()=>{}表示, 与Java略有不同.

委托

委托类似于C++中的函数指针, 委托可以保存与它签名相同, 返回类型也相同的方法. 调用委托时, 它将依次调用委托中留存的方法.

delegate关键字用于声明委托

泛型委托

事件

发布者类, 订阅者类交互可以使用事件来快速完成.
使用event可以为系统动态地, 模块化地添加关键事件回调方法.

event关键字用于声明事件, 事件与委托密不可分, 声明需要使用委托来定义事件类型

标准事件的声明为

传参部分, EventArgs是不能传输数据的, 如果需要传输数据, 那么需要使用继承于它的类, 声明事件时也需要使用泛型EventArgs.

如果需要移除在使用后移除事件, 和委托一样, 使用-=来移除委托.

事件中,-=和+=是事件允许的唯一运算符.

溢出检测运算符

cheked和unchecked都用于检测对应的块中是否有溢出

使用 is 运算符来避免盲目尝试类型转换

使用 as 运算符来避免强制转换为接口时可能抛出的异常

as运算符在一个类不能转型为指定接口时返回null, 而强制转换导致错误时将会抛出异常

Java的通配符 -> C#的Where子句

C#的约束允许 5 种类型

  • 类名

  • 接口名

  • class

    任何类, 数组, 委托, 接口

  • struct

    任何值类型

  • new()

    任何带有无参公共构造函数的类型
